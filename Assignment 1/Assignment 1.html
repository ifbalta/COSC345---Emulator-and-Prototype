Smartwatch prototype

Our smartwatch application will be the classic ball-in-a-maze game.


"Round maze" by Annielogue - Own work. Licensed under CC BY-SA 3.0 via Wikimedia Commons - http//commons.wikimedia.org/wiki/FileRound_maze.jpg#mediaviewer/FileRound_maze.jpg

The aim of the game is to get the ball to the center of the maze. The ball will be controlled by tilting the smartwatch in whichever direction the user wants to go. The game will have multiple levels of increasing difficulty. The game will have no scoring system. For our implementation, the maze will be a square.

There are many maze apps available for iOS, Android and Windows Phone. The smartwatch app marketplace is still very small, so there aren't many game apps yet. We feel that since maze games are very common, it will be easy for us to research the best way to implement our game, especially since none of us have any experience with game development or JavaScript.

The smartwatch application will be built with JavaScript and tested in the browser using our own smartwatch emulator. The emulator will be also be built with Javascript. None of us are proficient in Javascript and all of us will need to take a crash course. All of us are familiar and comfortable with Java.

Tools

We will use GitHub for version control. GitHub is widely used and comes with its own shell and GUI. The GUI is important for us because half the group has never used version control before and the GUI will let us get ahead of the learning curve. If we need extra functionality, then the Git shell is also available. We will need GitHub to commit our changes to the smartwatch emulator and our maze app.

Eclipse is one of the more widely used Java IDEs and has plug-in support for SQL databases, version control, Android or any other available library. Eclipse is also very good for large projects because it can search through multiple files very quickly, makes it easy to refactor code, and has plug-in support for Git. Git support is important to us because half the team is not familiar with Git and we would like to make the learning process easier.

The smartwatch emulator will be built with JavaScript and Dart. We like Dart because it looks very similar to Java-style code. The whole team is comfortable with the Java environment and we feel that we can work and learn JavaScript faster if we work in an environment that looks similar to Java. However, most of the work will be done in JavaScript because we know Dart has some underlying tech like the Dart VM and a special JavaScript interop library to wrap around other JavaScript libraries that might make over complicate our project.

We will also use HTML5 and CSS to build the web page that our emulator will run on.

We will use Chrome DevTools because it is convenient. We only need to right click and inspect element, the Sources panel lets you see all the scripts that are part of the inspected page. Unlike the page source panel , it gives you a “pretty print ”view. As long as you have Chrome, you have Chrome DevTools. There is no need to install anything. Debugging is always painful, but by using Chrome DevTools, it will clearly point out the problem and if we want to view any specific part on the page(like a button, text), we do not need to looking for this specific part in our long paragraph code. As long as we right click at the specific part of the page, Chrome DevTools will lead you to the corresponding code. There are also Developer Tools available on Internet Explorer, Safari and Firefox when it's time to ensure that our emulator is compatible to most other major browsers.

Smart Watch Emulator

Team Isabel Baltazar Lennox Huang

The smartwatch emulator will be written in JavaScript and other JavaScript libraries that may help us, such as EaselJS and BonsaiJS, and Bootstrap. We will use the Eclipse IDE thanks to its JavaScript development tools and GitHub because of its GUI and Git Shell. We estimate that it will take as us 6 weeks to learn the basics and best practices of JavaScript and 1 week to plan and design the beta version of the emulator. We expect that the team will continue researching and experimenting with JavaScript libraries while designing, implementing and testing the emulator.

We have found a JavaScript iPad emulator built three years ago which had roughly 2700 lines of JavaScript, not including extended jQuery libraries and around 1600 lines of CSS. They did not use HTML.

We estimate that our team will build an emulator with around 1500-2000 lines of JavaScript code, and around 1000-1500 lines of CSS and HTML, depending on whether or not they choose to emulate the gyroscope or use detailed graphics. Unlike the iPad emulator, our team does not intend to use only JavaScript to build the emulator as HTML5 has better graphics support than it did three years ago, and we expect to use frameworks like Bootstrap to lighten the HTML and CSS load.

Our team has only two years of programming experience and very little JavaScript expertise. We expect that the team will start slowly as they learn the best practices of JavaScript and its libraries and frameworks. We have allocated 7 weeks for the implementation of the smartwatch emulator and we expect that at least 2 of those weeks will be spent learning and setting up the various JavaScript graphics libraries.

We expect the completed smartwatch to have the following features
  • Basic touch gestures such as swiping and tapping
  • A basic clock app
  • Be able to run JavaScript smartwatch applications by other developers
  • A hardware button that can be utilized by other applications
Optional features
  • A gyroscope
  • A speaker
  • Be able to emulate the appearance of a Microsoft Band, Apple Watch, Pebble Watch and Samsung Gear.
