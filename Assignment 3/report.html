For the assignment 3, we redo the maze game simulator structure and rewrite the JavaScript.We use the physics 2D engine for the ball movement in the maze.We used several different images to be the maze game structure such as the ball, wall and the maze background.

Isabel Baltazar and Lennox Huang were assigned the code on the emulator and report, Raeciel Reyes and Kimi Zhou were learn the physics 2D engine structure to manipulates the ball can work smoothly in every level of the maze game.

For our application, it is a maze game and we use up, down, left and right on keyboard to control the ball movement. There are different levels in this game, the user will go to the next level once the ball arrive the hole.

Evaluation


The previous emulator and maze game application failed to keep the maze game application and browser independent of each other. The original maze game application used jQuery to handle user input, even though the maze game isn’t allowed to know about the browser. The emulator was supposed to be an interface between the application and the browser, but the emulator isn’t allowed to know anything specific about the application either.


We’ve rebuilt the emulator to have functions that raises flags to indicate user input. The Maze Game Application has been rewritten to use the Phaser.io framework and the emulator’s input flags. The emulator no longer knows about the game, and the game no longer knows about the browser.

Installation

Requires git VCS be pre-installed.
git can be downloaded from git-scm.com/downloads.

For compatibility with non-Safari browsers, node.js and Express.js or python must be installed.

If using a python server:

  • Install python from their website
  • Navigate to the webRoot directory and run python -m SimpleHTTPServer 8080 on the command line.
If using a node server:
  • Install node.js from their website
  • Install Express.js by running npm install express --save on the command line
  • Navigate to the webRoot directory, and start the server using the command node bootup
The emulator is open-source and available on GitHub. Get the code here.
  1. Clone the repository using git clone https://github.com/ifbalta/COSC345---Emulator-and-Prototype.git or Git clients such as SourceTree or the GitHub GUI
  2. Using Safari, open emulator.html in the broswer
  3. Using Google Chrome,
    • Using node, start server through node bootup and navigate to http://localhost:3000 in the browser
    • Using python, start server through python -m SimpleHTTPServer 8080 and navigate to http://localhost:8080

For Users

Starting up the emulator is as simple as opening emulator.html in Safari, and selecting an app from the menu.

Not using Safari?

The emulator is compatible with Google Chrome, Firefox, Internet Explorer and Opera if you have node.js and Express.js pre-installed.

For Developers

The emulator uses jQuery's getScript() function to retrieve AppObjects from other JavaScript files.

Installing your own application

  1. Applications must be written in JavaScript and follow the AppObject structure
  2. Store your application in the appDir/ directory.
    Your folder must have the same name as your main JavaScript file. If your file is named example.js it must be stored in the example directory.
    The resulting file path should be appDir/example/example.js.
  3. Add your application to the app tray on emulator.html. The app tray is a select menu with an label property. The value attribute must have the same name as your main JavaScript file.
    If your file is named example.js, then its value must also be example.js
The emulator expects an AppObject with three properties:

  • startUp

    should initialize the application.

  • stopScript

    should stop the application.

  • clear

    should clear the screen.

The emulator itself has only four functions:

  • bootup()

    @param filename
    the file from which the AppObject can be retrieved

    bootup() is tasked with clearing and stopping any previously running applications before initializing a new AppObject.

  • clear()

    clear() clears the emulator screen. It does not stop the application. Stopping the application should be the AppObject's stopScript property's responsibility.

  • A function linked to emulator.html's start button.

    The function uses jQuery to select a file from the app tray.

  • selectFile()

    @param filename the selected application

    Responsible for adding the file path to the file name.

Bugs
