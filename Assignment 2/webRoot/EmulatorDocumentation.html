Smart Watch Emulator Manual

Authors:
Isabel Baltazar
Lennox Huang

Language: JavaScript
Dependencies: jQuery version 1.11.1 provided by Google's hosted library.

The emulator acts like a generic smart watch and is not meant to mimic any currently available smart watch such as the Apple Watch or Moto 360. The emulator is written in JavaScript and jQuery and is run on the browser. It is a compatibility layer between the browser and smart watch application.
The emulator is compatible with Safari.
The emulator is compatible with Chrome, Firefox, Internet Explorer, and Opera if node.js and Express.js or python is pre-installed.

Features

  • A 320 x 320 screen
  • An app tray

Installation

Requires git VCS be pre-installed.
git can be downloaded from git-scm.com/downloads.

For Google Chrome compatibility, node.js and Express.js or python must be installed.

If using a python server:

  1. Navigate to the webRoot directory and run python -m SimpleHTTPServer 8080 on the command line.
If using a node server:
  • Install node.js from their website
  • Install Express.js by running npm install express --save on the command line
  • Navigate to the webRoot directory, and start the server using the command node bootup
The emulator is open-source and publically available on GitHub. Get the code here.
  1. Clone the repository using git clone https://github.com/ifbalta/COSC345---Emulator-and-Prototype.git or Git clients such as SourceTree or the GitHub GUI
  2. Using Safari, open emulator.html in the broswer
  3. Using Google Chrome,
    • Using node, start server through node bootup and navigate to http://localhost:3000 in the browser
    • Using python, start server through python -m SimpleHTTPServer 8080 and navigate to http://localhost:8080

For Users

Starting up the emulator is as simple as opening emulator.html in Safari, and selecting an app from the menu.

Not using Safari?

The emulator is compatible with Google Chrome, Firefox, Internet Explorer and Opera if you have node.js and Express.js pre-installed.

For Developers

The emulator uses jQuery's getScript() function to retrieve AppObjects from other JavaScript files.

Installing your own application

  1. Applications must be written in JavaScript and follow the AppObject structure
  2. Store your application in the appDir/ directory.
    Your folder must have the same name as your main JavaScript file. If your file is named example.js it must be stored in the example directory.
    Your application's file path should follow the appDir/example/example.js format.
  3. Add your application to the app tray on emulator.html. The app tray is a select menu with an label property. The label must have the same name as your main JavaScript file.
    If your file is named example.js, then its label must also be example.js
The emulator expects an AppObject with three properties:

  • startUp

    should initialize the application.

  • stopScript

    should stop the application.

  • clear

    should clear the screen.

The emulator itself has only four functions:

  • bootup()

    @param filename
    the file from which the AppObject can be retrieved

    bootup() is tasked with clearing and stopping any previously running applications before initializing a new AppObject.

  • clear()

    clear() clears the emulator screen. It does not stop the application. Stopping the application should be the AppObject's stopScript property's responsibility.

  • A function linked to emulator.html's start button.

    The function uses jQuery to select a file from the app tray.

  • selectFile()

    @param filename the selected application

    Responsible for adding the file path to the file name.

Bugs

  • The emulator cannot completely stop an application from running
  • The emulator does not always clear the screen
  • The emulator does not focus on the canvas when an application is running
